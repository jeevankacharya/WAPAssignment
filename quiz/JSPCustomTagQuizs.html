2. How do custom tags relate to JSTL?

Ans : JSTL stands for Java server pages standard tag library, and it is a collection of custom JSP tag libraries that provide common web development functionality. Similary Custom Tag is a user defined langauge element which provides functionlaity same as JSTL



3. What is the role of the URI in the TLD and the taglib directive?

Ans : The uri element in TLD is a unique name for tag library.


4. What is a tag handler class?

Ans : Tag handler class is the one which extends tag support framework class 

-– javax.servlet.jsp.tagext.TagSupport

A tag handler has access to the api that allows it to communicate with the JSP pages.


5. What is the role of attribute setters in a tag handler class?

Ans:  Main role of tag attribute is to pass information to the tag handler and customize its behaviour.


6. What is the role of the doTag() method in a tag handler class?

Ans: All tag logic,iterations and body evaluations oocur in this method.


7.What does the operation getJspContext().getOut().write(“Hi Bob”) do when called in a doTag() method?

Ans :The doTag() method takes the current JSP context object using the getJSPContext() method and uses it to send "Hi Bob" to the current JSPWriter Object.


8. What does the operation getJspBody().invoke(null) do when called in a doTag() method?  

Ans : When getJspBody().invoke(null) is called, it is the output resulting  from the execution of the tag body's JSP content that gets passes to the client, not the JSP code itself.
